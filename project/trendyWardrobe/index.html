Trendy Wardrobe

A solution for anyone trying to simplify their morning!

The Trendy Wardrobe is an innovative system that recommends what to wear everyday given data like the user's calendar, preferences and current weather. This initiative combines an image capturing system, image detection and classification powered by Amazon Web Services, SVM machine learning and Online API's as well as an user friendly app that helps users decide what the appropiate clothing item is for the day.

Motivation

Motivation

We live in an era where data is everything. Everyday, we hear new milestones achieved thanks to data analytics and machine learning. Humans and machines are getting more intelligent everyday. Time is something that has not changed. Mark Zuckerberg said in an interview that he wears the same grey clothes everyday in order to focus his energy towards improving the community. The main motivation for our project was to build a system that simplifies the day to day morning routine by helping people choose what to wear by giving a recommendation that is personalized for each user.

Feature Sets

Image Classification for Detecting Clothes Location Change

– Motion sensor detects clothes movement and triggers the camera to capture an image and send to the server.
– The server analyzes this image by Google Cloud Vision API and SVM, and recognizes the clothes id.
– Update this clothes’s location(In wardrobe or not in wardrobe) into MongoDB.

Clothes Recommendation

– Only choose the clothes that are in the wardrobe.
– According to users’ preferences, users’ schedules and weather, provide recommendation for those clothes in the wardrobe.
– Demonstrate by Android application.

Image Classification for Detecting Clothes Location Change

After receiving an image from the board, the next step in the is to clasify it. The image is processed through the Google Cloud Vision API in order to find features like dominant colors, type of weather the clothes should be used in, formality of the clothes and brightness of the clothes. The data is normalized and sent to an SVM Module to get an unique id for each piece of clothing. Such clothes id is given to our clothes’ inventory database in order to notify the location change of this clothing. If this clothing is in the wardrobe before, we will change it to "out of wardrobe". Otherwise, this clothing's location will be changed from "out of wardrobe" to "in wardrobe".

Recommendation System

In our recommendation system, we need four information to feed into our SVM learning system. The first one is the clothes’ inventory. All the clothes information including location(in wardrobe or not) is stored in the MongoDB in the AWS. By query MongoDB, we can know how many clothes that we can choose now since we only recommend the clothes that are in the wardrobe. The second one is users' preference. Users can set up their preference in our Android application “USER PREFERENCE” page. Now we provide one option for users to choose. A user can choose if he/she prefers short-sleeve or not. The third one is users' schedule. We link our Android application to Google Calendar API. By getting the user’s everyday event, we can recommend he/she the clothes which is suitable their event. For example, the user should wear more official while he/she has a meeting event. The last one is the weather information. Server will get this information automatically by using Openweathermap API. Therefore, we can know that we should wear a jacket while it’s cold.

After getting those four information, we feed them to our SVM learning system. Then it will output one recommended clothes. For our SVM training system, we already used more than 1000 data to train it before. Therefore, it can provide very accurate recommendation according to our input information.

At the end of the day, users can feedback their feeling about our recommended clothes. And those can make our training system better.

Android Application

This is used to demonstrate our recommendation system. There are five functions in our Android application. The first one is the “USER PREFERENCE” page which is let users set up their preference. The second one is the “Show Clothes” page which is let users know their clothes inventory, including the clothes location(in wardrobe or not). The third one is our main function “RECOMMENDATION”. Users can get recommendation everyday from this page. The fourth one is the “USER FEEDBACK” page, which provide a place for users to feedback their opinion about the recommendation. The last one is for the demo only.
